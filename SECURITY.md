# Security Policy

## Supported Version

Security fixes are applied to the latest commit on the `main` branch.

## Reporting a Vulnerability

Do not disclose credentials, personal data, or exploitable details in a public issue.

Use GitHub's private vulnerability reporting feature from the repository's **Security** tab. Include the affected route or file, reproduction steps, impact, and any suggested mitigation.

For non-sensitive defects and feature requests, use the public issue tracker.

## Repository Data

Production content, uploaded media, environment files, and private keys are intentionally excluded from this public repository. The committed JSON file contains fictional demo data only.
