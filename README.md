# Театральная Завалинка

Готовый к запуску сайт фестиваля на `HTML/CSS/JS` с лёгкой админкой на `Node.js + Express`.

## Что внутри

- 4 публичные страницы: главная, о фестивале, документы, контакты.
- Адаптивный интерфейс без Google Sites и тяжёлых фреймворков.
- Простая админка на `/admin` с логином и паролем.
- Хранение контента в `data/site-content.json`.
- Загрузка документов в `public/uploads/docs`.
- Загрузка изображений в `public/uploads/images`.

## Требования

- Node.js 18+.
- npm 9+.

## Установка

```bash
npm install
```

## Настройка окружения

1. Скопируйте `.env.example` в `.env`.
2. Задайте свои значения:

```env
PORT=3000
SESSION_SECRET=change-me-please
ADMIN_USERNAME=manager
ADMIN_PASSWORD=change-me-too
PUBLIC_SITE_URL=http://localhost:3000
```

## Запуск

Для обычного запуска:

```bash
npm start
```

Для разработки:

```bash
npm run dev
```

После старта сайт будет доступен по адресу:

- `http://localhost:3000/home`
- `http://localhost:3000/info`
- `http://localhost:3000/docs`
- `http://localhost:3000/contacts`
- `http://localhost:3000/admin`

## Вход в админку

- Адрес: `http://localhost:3000/admin`
- Логин и пароль берутся из `.env`

## Как менеджеру редактировать сайт

### Тексты и ссылки

1. Откройте `/admin`.
2. Войдите под логином менеджера.
3. Перейдите в нужный раздел:
   - `Основное` для общих ссылок и реквизитов,
   - `Главная` для hero-блока, этапов и фактов,
   - `О фестивале` для описания, статусов и видео,
   - `Контакты` для контактной информации,
   - `Карточки` для жюри, экспертов, гостей, партнёров и победителей,
   - `Документы` для документов по годам.
4. Нажмите `Сохранить изменения`.

### Документы

- Документы можно добавить как внешнюю ссылку или загрузить файл прямо из админки.
- Загруженные файлы сохраняются в `public/uploads/docs`.
- Для локального файла нажмите `Загрузить` в карточке документа, затем `Сохранить изменения`.

### Изображения карточек

- Изображения для жюри, экспертов, гостей, партнёров, победителей и контактных лиц загружаются через админку.
- Загруженные картинки сохраняются в `public/uploads/images`.

### Порядок элементов

- В каждом списке есть кнопки `Вверх` и `Вниз`.
- После перестановки нужно нажать `Сохранить изменения`.

## Где лежат данные

- Весь редактируемый контент: `data/site-content.json`
- Документы: `public/uploads/docs`
- Изображения: `public/uploads/images`

## Примечание

Публичные страницы рендерятся сервером на основе `data/site-content.json`, поэтому после сохранения в админке изменения сразу появляются на сайте без пересборки фронтенда.
